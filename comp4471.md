## What the Report Says We Are Doing

### Problem
DA3 (Depth Anything V3) was trained on terrestrial data. Underwater it fails because:
- Red light absorbed within ~3 m → blue-green colour cast
- Backscatter from suspended particles reduces contrast
- Result: wrong metric scale, noisy predictions in low-contrast regions

### Goal
Fine-tune DA3 Mono Metric Large to work accurately underwater for AUV competitions (SAUVC, RoboSub, RoboTX) — detecting gates, flares, buckets and estimating their distance from a single camera.

### Full System Pipeline
```
Camera feed
    → Image Preprocessing (Gray World WB + Histogram Stretch)
        → [Branch A] YOLOv26-seg (TensorRT)  →  instance masks
        → [Branch B] DA3 Mono Metric Large (TensorRT)  →  depth map
            → Mask-Depth Fusion
                → median depth per object
                    → "Gate at 2.3 m"  (ROS 2 topic)
```

## Files Created — What Each Does

### `train/lora.py`
Injects LoRA adapters into DA3's DINOv2 backbone.

**Key functions:**
- `inject_lora(model, rank=8, alpha=16.0)` — walks every `Attention` block, replaces `qkv` and `proj` Linear layers with `LoRALinear` wrappers
- `freeze_backbone(model)` — freezes all backbone weights except LoRA A/B matrices; DualDPT head stays fully trainable
- `count_trainable(model)` — prints how many params are trainable vs total (~0.5–2%)
- `merge_lora_weights(model)` — bakes LoRA delta into base weights (for TensorRT export)

**How LoRA works:**
```
W_adapted = W_frozen + B @ A * (alpha / rank)
A: (in_features, rank)   ← init kaiming
B: (rank, out_features)  ← init zeros (so delta=0 at start)
```

---

### `train/losses.py`
Two losses combined into `UnderwaterDepthLoss`.

**SILog loss** (penalises depth shape, tolerates global scale offset):
```
g_i = log(pred_i) - log(gt_i)
L_silog = mean(g^2) - 0.85 * mean(g)^2
```

**Gradient loss** (Sobel filter, preserves sharp edges at object boundaries):
```
L_grad = mean(|∇x_pred - ∇x_gt| + |∇y_pred - ∇y_gt|)
```

**Combined:**
```
L_total = L_silog + 0.5 * L_grad
```

---

### `train/dataset.py`
Loads Unity paired RGB + depth frames. Applies preprocessing at load time (same at train AND inference).

**Preprocessing pipeline:**
1. Gray World white balance — rescales attenuated red channel, suppresses blue-green
2. Percentile histogram stretching (2nd–98th percentile per channel)

**Depth format supported:**
- `.npy` — float32 array, values in metres (preferred from Unity)
- `.png` — uint16 PNG, values in millimetres (divide by 1000 to get metres)

**Auto train/val split:** 80/20 by default, reproducible with fixed seed.

---

### `train/train.py`
Main training script. Run this when Unity data is ready.

```bash
python train/train.py \
    --data_root data/unity \
    --model_name da3metric-large \
    --epochs 30 \
    --batch_size 4 \
    --lora_rank 8 \
    --lora_alpha 16.0 \
    --lr 1e-4
```

**What it does:**
1. Downloads `da3metric-large` pretrained weights from HuggingFace
2. Injects LoRA, freezes backbone
3. Trains with AdamW + CosineAnnealingLR
4. Validates each epoch, reports AbsRel / RMSE / δ<1.25
5. Saves `checkpoints/best.pth` (best AbsRel) and `checkpoints/last.pth`
6. Supports `--resume checkpoints/last.pth` to continue from a checkpoint

---

### `train/evaluate.py`
Benchmarking script. Compare baseline vs fine-tuned, or test on public datasets.

```bash
# Zero-shot baseline
python train/evaluate.py --data_root data/unity/test

# Fine-tuned model
python train/evaluate.py --data_root data/unity/test \
    --checkpoint checkpoints/best.pth

# FLSea or SQUID benchmark (just point at their folder)
python train/evaluate.py --data_root data/flsea \
    --checkpoint checkpoints/best.pth
```

**Metrics reported:**
| Metric | Meaning | Lower is better |
|--------|---------|----------------|
| AbsRel | Mean absolute relative error | Yes |
| SqRel | Mean squared relative error | Yes |
| RMSE | Root mean squared error (metres) | Yes |
| RMSE_log | RMSE in log depth space | Yes |
| δ<1.25 | % pixels with max(pred/gt, gt/pred) < 1.25 | No (higher better) |
| δ<1.25² | Same with threshold 1.5625 | No |
| δ<1.25³ | Same with threshold 1.953 | No |
